### Warum PWA leer war
- Filter im Backend: `state = "assigned"` — nur Pickings die bereit und nicht abgeschlossen sind
- Manuell erstellte Transfers waren entweder sofort validiert (`done`) oder ohne Lagerbestand (`confirmed`)
- Pickings müssen den Flow durchlaufen: `draft → action_confirm → action_assign → assigned`

### Quality Alert — Bugs & Fixes
- **Bug 1:** `api_create_alert() takes 2 positional arguments but 3 were given`
  - Ursache: `call_method` baute `[[], vals]` → 3 Args. `@api.model`-Methoden brauchen `[vals]`.
  - Fix: `quality.py` Router auf `odoo.execute_kw("quality.alert.custom", "api_create_alert", [vals])` umgestellt.
- **Bug 2:** `You are not allowed to create 'Quality Alert' records`
  - Ursache: Odoo-Access-Rights — admin war nicht in `Qualitäts-Manager`/`Qualitäts-Nutzer` Gruppe.
  - Fix: `api_create_alert` nutzt jetzt `self.sudo().create(vals)` → Rechteprüfung auf API-Ebene (FastAPI), nicht Odoo-Ebene.
- **Status:** ✅ Quality Alert ohne Foto funktioniert

### Quality Alert: Doppelklick-Bug & Timeout
- **Problem:** Absenden-Button reagierte nicht sofort (Foto-Upload dauerte zu lange). Mehrfaches Klicken erzeugte ~14 doppelte Alerts.
- **Ursache 1:** `httpx.ReadTimeout` — Odoo JSON-RPC Timeout war 30s, Foto-Upload (Base64 über JSON-RPC) braucht länger.
- **Ursache 2:** Kein Doppelklick-Schutz auf dem Submit-Button.
- **Fix 1:** `odoo_client.py` — Timeout von 30s auf 120s erhöht.
- **Fix 2:** `app.js` — Submit-Button wird nach erstem Klick deaktiviert (`disabled = true`) und zeigt "Wird gesendet…". Erst bei Fehler wird er reaktiviert.
- **Status:** ✅ Gefixt

### Voice-Picking: Vosk → Whisper Migration
- **Migration komplett:** Vosk (WebSocket, Kaldi, ~15-20% WER) ersetzt durch Whisper (REST, faster_whisper small, ~8-10% WER)
- Neuer STT-Client: `backend/app/services/whisper_client.py` (REST statt WebSocket)
- Docker-Compose: `vosk` Service → `whisper` Service (`onerahmet/openai-whisper-asr-webservice`)
- Backend konvertiert WebM→WAV weil Whisper-Container minimales ffmpeg hat

### Voice-Picking: PTT → Voice-Toggle-Modus
- **Push-to-Talk** durch **Voice-Toggle** ersetzt (Mic-Button oder Taste M)
- Automatische Silence Detection: RMS-basiert (Schwelle 25), 700ms Stille → senden
- TTS-Feedback-Loop-Schutz: Mikrofon wird stumm geschaltet während TTS spricht
- setInterval(30ms) statt requestAnimationFrame für zuverlässiges Audio-Monitoring
- M-Taste: repeat-Guard gegen gedrückt-halten-Problem
- Betroffene Dateien: `pwa/js/voice.js` (Rewrite), `pwa/js/app.js` (Voice-Toggle-Integration)

### Voice-Picking: Gelöste Probleme
1. Whisper `medium` auf CPU: Timeout >2min → `faster_whisper` + `small` Modell (~1-2s)
2. WebM-Dekodierung im Whisper-Container fehlgeschlagen → Backend-seitige WAV-Konvertierung
3. Silence Detection: Schwellwert zu niedrig (15, Rauschen bei 17-33) → fester Wert 25
4. Kalibrierung durch TTS-Audio korrumpiert → Kalibrierung entfernt, fester Schwellwert
5. Wortanfänge verloren → Recording sofort starten statt nach Speech-Detection
6. TTS Feedback-Loop → Mikrofon-Tracks stummschalten während TTS
7. M-Taste Repeat → `e.repeat` Guard

### Voice-Picking: Bekannte Einschränkungen
- Round-Trip ~2-4s — funktioniert, könnte aber reaktiver sein
- M-Taste: Voice-Modus bleibt aktiv wenn Buttons versteckt (Seitenwechsel)
