## How we worked

- First we stabilized the infrastructure and Odoo access before trusting any feature work.
- Then we used a clean local test database `picking` to verify the backend flow without legacy data issues.
- After the API path was stable, we restored the real `masterfischer` backup in parallel so no working state was lost.
- Each major step was checked with logs, direct Odoo calls, `curl`, and `test-api.py`.
- We documented the important decisions and fixes directly in Obsidian while working.

## Important decisions from today

- Odoo admin access is done directly via `http://localhost:8069/`.
- The `/odoo/` proxy path is not treated as the reliable admin path in this PoC.
- The backend currently talks to Odoo over JSON-RPC.
- For the current Odoo 18 setup, the picking completion logic is based on `stock.move.picked`.
- The active business database is now `masterfischer`.
- The old `picking` database remains as a fallback and reference environment.
